# Why I rebuilt StrafeKit from scratch
Studying for the OSCP taught me that tool organization is half the battle.
When the framework I built to solve that problem became a problem itself —
I decided to rebuild it properly and learn Python along the way.

<!-- more -->
I was studying for the OSCP and wanted a way to organize my engagements.
Manual enumeration, organization and recording of the engagement grew to be
a bottleneck and I felt I spent more time looking and organizing data more
than applying what I found.

Initially I used an AI to slowly build the features I wanted but as time
went on the functionality of the framework grew inconsistent. The codebase
was a mess and I then spent more time on tokens and waiting for AI output.
Don't get me wrong — it was a huge step up from manual enumeration and
typing commands, and when it worked it worked great. But I wanted the
framework to be more consistent.

I figured if I was going to rebuild the framework this would be a great
opportunity to learn Python and industry infrastructure practices.
